# Aeris Engineering Rules

Product Goal

Answer:

"Am I getting faster and fitter over time?"

Constraints

- Single user only
- No auth
- No multi tenancy
- No vector database
- No coaching recommendations
- No training plans

Tech Stack

- Next.js
- TypeScript
- Supabase
- Claude
- Recharts
- Tailwind

Coding Standards

- Strict TypeScript
- Zod validation
- Vitest
- Playwright
- Repository pattern

TDD Rules

Never write production code before a failing test exists.

Workflow:

Red
Green
Refactor

Definition of Done

- Tests pass
- Build passes
- Types pass
- Acceptance criteria met
